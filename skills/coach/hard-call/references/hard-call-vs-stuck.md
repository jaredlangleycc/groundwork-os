# Hard Call vs. Stuck

How to distinguish a genuine hard call (use hard-call posture) from a user who is stuck and needs business-coach. Wrong posture wastes the user's time at best, alienates them at worst.

## What stuck looks like

The user has a decision in front of them. They have not made it. They want help making it.

**Symptoms of stuck:**

- First mention of the decision (this is a new question)
- One of the paths is obviously better when you ask the right questions
- The user is missing information they can go gather (a number, a conversation, a market signal)
- The user has not yet framed the decision precisely
- The user has not weighed the options against their stated values
- Stress is about complexity, not weight

**Posture for stuck:** `business-coach`. Open questions, clarifiers, draft using their words, hold to their stated bar. Once the user has the missing piece, the decision becomes obvious or close to obvious.

## What a hard call looks like

The user has been sitting with the decision. Both paths are defensible. Information-gathering has been done. The friction is the choice itself, not the inputs.

**Symptoms of hard call:**

- Second, third, or fourth mention of the same decision
- Both paths have already been analyzed and have credible cases
- The user can articulate the strongest argument for each side
- Additional information would not change the call
- The decision is about identity, values, or trade-offs the user cares about deeply
- Stress is about weight, not complexity
- The user has been physically holding it (sleeping poorly, distracted, snippier than usual)

**Posture for hard call:** `hard-call`. Hold the ambiguity. Surface what the user is avoiding naming. Use the relief test. Do NOT recommend a path.

## Diagnostic questions

Run these 4 questions when the user surfaces a decision. The answers route to the right posture.

### Question 1: When did you start sitting with this?

- "Just today" or "this week" leads to likely STUCK
- "A few weeks ago" or "I keep coming back to it" leads to likely HARD CALL

### Question 2: If you had to pick the better path right now, could you?

- "Yes, but..." leads to likely STUCK (use business-coach to surface what changes with the "but")
- "No, both feel right" leads to likely HARD CALL
- "No, both feel wrong" leads to likely HARD CALL with deeper values question underneath

### Question 3: What would change your mind?

- A specific piece of data leads to likely STUCK (use business-coach to go get the data)
- A conversation with a specific person leads to likely STUCK (route to "have the conversation")
- "I don't know" or "Nothing would" leads to likely HARD CALL

### Question 4: What does the indecision cost you per week?

- "Nothing really" leads to likely STUCK (or fake-hard-call, more on this below)
- "Mental space, I keep re-thinking it" leads to likely HARD CALL
- "It's affecting my work / sleep / family" leads to likely HARD CALL

## Common mismatches

### Treating stuck as hard call

The user has not gathered information they could easily gather. You run the full hard-call sequence. They feel held but not helped. The right move was: "Go talk to [X]. Come back with the answer."

Cost of this mismatch: 30 minutes of false depth, user remains stuck.

### Treating hard call as stuck

The user has been sitting with a decision for 4 weeks. You run business-coach: open questions, clarifiers, drafts. The user feels processed but not held. The decision they have been avoiding remains avoided.

Cost of this mismatch: Worse than the first. The user concludes the system cannot hold the real stakes.

## Fake hard calls

Some "hard calls" are performances. The user is asking for permission, validation, or absolution, not coaching.

**Symptoms of a fake hard call:**

- The user keeps reframing the question to lead you toward one answer
- The user dismisses your questions when they do not lead where they want
- The user has already decided and wants you to confirm
- The relief test produces a clear answer but the user keeps deliberating
- The user is testing the relationship, not the decision

**Posture for fake hard calls:** Name it gently. "It sounds like you've decided. What's making it hard to claim?" If the user keeps performing, end the session. Schedule a revisit. The work to be done is not coaching, it is something else (often values clarification or a hard conversation with a third party).

## When stuck becomes hard call

Sometimes a decision starts as stuck and becomes a hard call after information-gathering. Example:

Week 1: "Should I launch a course or stay 1:1?" - STUCK because user has not validated demand for the course.

User goes out, does 10 interviews. 8 want the course, 2 don't.

Week 3: "Should I launch a course or stay 1:1?" - HARD CALL because both paths are now defensible. Demand is real for the course, but the user values the 1:1 work more than the scale.

Different posture in week 3 than week 1, same surface question. The diagnostic questions catch the shift.

## When hard call becomes stuck

The reverse also happens. Sometimes a hard call resolves when one of the underlying assumptions changes.

Example: "Quit day job or stay?" framed as a hard call. Through the hard-call session, it surfaces that the user assumed partner support without confirming. Partner conversation happens. Partner says "not until business hits $X for 3 months." 

The decision is no longer a hard call. It is now stuck on "what's the path to $X for 3 months?" which is a business-coach decision, not hard-call.

The skill switches when the underlying constraint changes.

## How to communicate the routing

When you have diagnosed (stuck or hard call), tell the user explicitly. The transparency builds trust.

For stuck:
> "This sounds like a normal decision that just feels hard because you have not made it yet. I will use the regular coach posture: open questions, then a draft. If it turns out to be a hard call, I will say so and switch."

For hard call:
> "This is a hard call. Both paths are defensible. I will hold this with you. I will not recommend a path. The work is going to feel slower than normal coaching, because it is."

Naming the posture lets the user calibrate their expectations. They know what kind of support they are getting.

## Edge case: the user disagrees with the routing

The user thinks it is a hard call, you diagnosed stuck. Or vice versa.

Defer to the user once. Run their preferred posture for one session. If it lands, you were wrong. If it does not land, the diagnostic was right, surface the gap: "We tried hard-call posture. The relief test produced a clear answer. Does that feel resolved, or are we actually in stuck territory needing more data?"

The user's read of the situation has signal. Trust it once, recalibrate if it does not work.
