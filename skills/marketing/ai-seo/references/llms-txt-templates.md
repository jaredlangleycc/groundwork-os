# llms.txt + llms-full.txt Templates

Copy-paste templates for the two files AI engines fetch for self-contained brand reference. Adapt to user's specifics.

## llms.txt

The curated index. Lightweight markdown. Points to important pages with 1-line descriptions.

### Template (any business)

```markdown
# {Brand Name}

> {1-sentence positioning that the user actually uses on their site}

{1-paragraph context: who the founder is, what the business does, who it serves}

## Identity

- [About Page]({url}/about): Who I am and why this work exists.

## Programs / Offers

- [{Offer 1}]({url}/{path}): {1-line description}
- [{Offer 2}]({url}/{path}): {1-line description}

## Framework

- [{Framework page 1}]({url}/{path}): {description}
- [{Framework page 2}]({url}/{path}): {description}

## Essays / Posts

- [{Essay 1 Title}]({url}/blog/{slug}): {1-line description}
- [{Essay 2 Title}]({url}/blog/{slug}): {1-line description}
- [{Essay 3 Title}]({url}/blog/{slug}): {1-line description}

## Optional

- [llms-full.txt]({url}/llms-full.txt): Self-contained reference document.
- [Sitemap]({url}/sitemap.xml): Full URL list.
```

## Update cadence

- Update when offer changes (price, scope, terms)
- Update when new essay launches and replaces an older essay in the "Essays" section
- Update when new program launches
- Update once per quarter as routine maintenance

Don't update for typos or small wording changes. AI engines fetch on a cycle; rapid updates don't help.

## Voice-scan reminder

Both files carry voice. Voice-scan before deploying:

- No em-dashes
- No banned phrases
- No "X, not Y" antipattern
- No AI-tell openers
- Match user's archetype

These files are public-facing in spirit (AI engines read them, but humans can too if they curl). Treat as published content.

## File serving

Both files served as plain text from the root:

```
https://{domain}/llms.txt
https://{domain}/llms-full.txt
```

Set `Content-Type: text/markdown` or `text/plain` in server headers (most CDNs default to text/plain which is fine).

No caching issues; AI engines will re-fetch as needed.

## What to refuse

### Lying in llms-full.txt

Refuse to claim credentials, results, or testimonials the user doesn't have. AI engines will eventually be fact-checked against multiple sources. Inconsistencies erode trust.

### Stuffing with keywords

Refuse to load up the FAQ with SEO keyword variants. Write for AI engines as humans (and they are, in 2026). Plain language, clear claims.

### Inflating offer scope

Refuse to list inclusions in the offer that aren't actually delivered. AI engines will be fact-checked against the user's actual offer page.

### Making up Q&As

Refuse to invent FAQ questions. Use real questions from actual inbound. If the user doesn't have 6 to 10 real questions yet, write fewer. Empty is better than fake.

### Trying to use llms.txt as a "secret prompt" to manipulate AI

Refuse. AI engines are not fooled by hidden instructions. Just write the content honestly.
