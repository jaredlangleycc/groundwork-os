# Repurposing From Essay

Step-by-step: take one Sunday essay and produce 6 derivative social posts in 90 minutes. The spine pattern in execution.

## Prerequisite

The blog essay must be drafted. This skill doesn't generate the essay; it strips the essay into platform-shaped pieces. If the essay isn't done, the user is repurposing nothing.

The essay should:

- Be 1000 to 1500 words
- Have a clear thesis in the first 2 paragraphs
- Have 3 to 5 distinct beats (sections within the body)
- Have a clear takeaway or land
- Have at least one quotable line

If any of these are missing, the essay isn't ready. Push back to the user.

## The 90-minute workflow

### Step 1: Read the essay once (5 minutes)

No taking notes. No marking up. Just read the whole essay through. The first read is for pattern recognition.

### Step 2: Mark the parts (10 minutes)

Open the essay. Highlight or mark:

- [HOOK], the strongest hook-candidate line (often the first or second line)
- [BEAT 1], the first beat of the body (one paragraph or section)
- [BEAT 2], the second beat
- [BEAT 3], the third beat (if present)
- [LAND], the takeaway sentence(s)
- [QUOTE], the most quotable line
- [PROOF], any specific example or named person

You should end with 5 to 7 marked sections. If you can't mark them, the essay's structure is muddy. Hand back to the user.

### Step 3: Draft the Threads chain (15 minutes)

Convert the 5 marked sections into 5 Threads posts:

```
Post 1/5: [HOOK + 1 line of frame]
Post 2/5: [BEAT 1 condensed to 2-3 sentences]
Post 3/5: [BEAT 2 condensed]
Post 4/5: [BEAT 3 OR LAND condensed]
Post 5/5: [Asset link + reply prompt]
```

Each post under 500 characters. Use the hook bank if the essay's opening line doesn't translate to a punchy Threads opener.

Voice-scan.

### Step 4: Draft the LinkedIn personal long-form (10 minutes)

Pull the strongest hook line + first 2 beats + invite line.

Structure:

```
[HOOK LINE, 6 to 12 words]

[FRAME, 1 paragraph from essay opening, 60-100 words]

[BEAT 1 condensed, 100-150 words]

[BEAT 2 condensed, 100-150 words]

[LAND, 1-2 sentences]

[INVITE, Reply or DM prompt, 1 sentence]
```

Character count target: 1200 to 1800. Cut if over.

Voice-scan.

### Step 5: Draft the IG carousel (20 minutes)

8 slides. One idea per slide.

```
Slide 1: HOOK, bold claim, on visual
Slide 2: FRAME, context for the claim
Slide 3: BEAT 1, first idea
Slide 4: BEAT 2, second idea
Slide 5: BEAT 3, third idea (or expand a prior beat)
Slide 6: PATTERN, what these have in common
Slide 7: FIX, the takeaway
Slide 8: CTA, link in bio / take the lead-gen assessment / DM me / etc.
```

Each slide: 1 to 2 sentences max. Mobile-readable in 1 second per slide.

Visual: brand template, Archivo Black headlines, Manrope body, brand palette. Use the user's saved IG carousel template (don't redesign each post).

Voice-scan slide copy.

### Step 6: Draft the short-form video script (10 minutes)

75-second TikTok / Reels script:

```
[0-3 sec HOOK]
The boldest single line from the essay. No preamble.

[3-50 sec BEAT]
The strongest single beat. Pick the one that stands alone best.

[50-65 sec LAND]
The takeaway, condensed to 2 sentences.

[65-75 sec CTA]
"Follow for more on X" or "Read the full essay, link in bio."
```

Burned-in captions. Brand visual conventions per `platform-format-specs.md`.

Voice-scan.

### Step 7: Pull the quote graphic line (5 minutes)

Identify the single most quotable line from the essay.

Criteria:

- Stands alone without context
- 4 to 12 words
- Counterintuitive, declarative, or specific
- Doesn't require setup to land

Examples from a Langley essay:

- "Most of the time it's the second one."
- "The fix is the pause."
- "Reactive leadership feels productive. It isn't."
- "Your calendar is a confession."

Render in brand template. Single image post for Threads + IG.

### Step 8: Pull the LinkedIn company page carousel angle (15 minutes)

Either:

(a) Repost the IG carousel as-is to company page (lowest effort)

(b) Restructure into a framework-led carousel:

```
Slide 1: Framework name or claim
Slide 2: Defining the framework
Slide 3: Element 1
Slide 4: Element 2
Slide 5: Element 3
Slide 6: How to apply
Slide 7: CTA
```

Tone shift: less first-person, more "we observe" or "the pattern is."

Voice-scan.

## Time accounting

| Step | Time |
|---|---|
| 1. Read essay | 5 min |
| 2. Mark parts | 10 min |
| 3. Threads chain | 15 min |
| 4. LinkedIn long-form | 10 min |
| 5. IG carousel | 20 min |
| 6. TikTok script | 10 min |
| 7. Quote graphic | 5 min |
| 8. LinkedIn company carousel | 15 min |
| **Total** | **90 min** |

This is the discipline. 90 minutes, 6 to 8 derivative pieces.

## When the workflow takes longer than 90 minutes

Diagnose:

- **The essay is muddy.** Beats aren't clear. Hand back to the user.
- **The user is rewriting per platform instead of stripping.** Refuse the rewrite. Strip is the discipline.
- **The user is fighting voice-rules.** Either voice-rules needs an update, or the user is over-polishing.
- **The first essay of a new pillar.** Voice calibration takes time. Allow 120 minutes for the first 2 to 3 essays of a new pillar.

Don't allow the workflow to creep past 120 minutes in any week. The spine is supposed to free time, not consume it.

## What this workflow does NOT do

- Doesn't generate the essay (use `copywriting` or write it from scratch)
- Doesn't write blog posts (use blog template + `copywriting`)
- Doesn't replace one-off real-time posts (those are off-spine, written fresh)
- Doesn't auto-schedule (use platform native or Buffer)

## Output of the workflow

After 90 minutes, the user has:

1. Threads chain (5 posts)
2. LinkedIn personal post
3. IG carousel slide copy
4. TikTok script
5. Quote graphic line
6. LinkedIn company page carousel

Plus the existing:
- Blog essay (published Sunday)
- Newsletter (sent Sunday)

That's 8 distributable pieces for the week.

The scheduling itself happens separately (Friday afternoon block, ~30 min to push everything into the schedulers).

## Common failure modes

### "I'll just post the essay everywhere"

Refuse. Each platform's culture is different. Cross-posting identical text fails the algorithm on every platform.

### "Let me write each one fresh from scratch"

Refuse. The spine's value is repetition of the same idea across formats. Fresh-each-time = 6 hours, not 90 minutes.

### "The essay doesn't have a clear beat structure"

Don't strip. Hand back to the user to revise the essay first.

### "I want to add new ideas in each derivative"

Refuse for derivative pieces. Off-spine posts can have new ideas. Spine derivatives are the same idea, different shape.

### "This is taking too long"

Set a hard timer. 90 minutes total. If a single step blows past its slot, move on with the draft you have. Voice-scan catches the rough spots.

## Quality bar

Even at 90 minutes, the bar:

- Every derivative gets voice-scanned
- No derivative reuses identical first-line hook
- Each platform's format spec is respected
- Brand visual conventions hold

If a derivative doesn't meet the bar at 90 minutes, ship it anyway and revise next week. Don't burn the spine on perfection theater.
